# Lessons

## 2026-03-10
- Bootstrap the repo workflow first: create `tasks/todo.md` and `tasks/lessons.md` before editing the skill so planning and review are tracked from the first pass.
- When extracting from `deep-research`, keep durable state and verification discipline, but defer presentation artifacts until the search-control loop is stable.
- For CSV-first search pipelines, keep the control plane in a node ledger and keep evidence/cache rows separate; do not mix pruning state with result payloads in one wide table.
- If the goal is a convincing first skill, stop at a shallow loop: `search_nodes.csv`, `child_query_cache.csv`, `artifact_rows.csv`, and `report.md` are enough.
- When complexity grows, prefer escalating to `$deep-research` rather than growing recursive layers or orchestration inside this skill.
- Global installation should copy only the essential skill files, not project-only folders like `tasks/` or `demo/`.
- For theme-led humanities runs, avoid collapsing the report into one canonical theory; pair optical mirror sources with relational mirroring and a philosophical baseline.
- Put `(node/cache/artifact)` references directly into `report.md` when the goal is an auditable exploratory search rather than a polished essay.
- If a mounted drive has a broken parent `.git`, stop git discovery with a project-local repo and avoid relying on the parent tree for release work.
